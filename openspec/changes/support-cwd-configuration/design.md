## Context

当前项目使用 TypeScript 和 Node.js 22 开发，配置文件主要用于定义任务和工作流。目前配置文件不支持 cwd 配置，所有任务默认在当前工作目录下执行。这限制了任务的灵活性，特别是当任务需要在不同的目录中执行时。

## Goals / Non-Goals

**Goals:**
- 支持在配置文件顶层声明项目级别的 cwd
- 支持在单个任务中声明独立的 cwd
- 任务级别的 cwd 优先于项目级别的 cwd
- 保持向后兼容，未声明 cwd 时使用默认行为

**Non-Goals:**
- 不支持相对路径的自动解析（用户需提供绝对路径或明确相对路径）
- 不修改现有的任务执行引擎核心逻辑

## Decisions

1. **配置文件结构**
   - 在顶层添加 `cwd` 字段（可选）
   - 在每个任务对象中添加 `cwd` 字段（可选）
   - 使用字符串类型表示路径

2. **优先级规则**
   - 任务级 `cwd` > 项目级 `cwd` > 进程当前目录
   - 这样设计确保任务可以独立覆盖项目级别的设置

3. **Schema 更新**
   - 使用 Zod 或类似验证库更新配置 schema
   - cwd 字段标记为可选且为字符串类型

4. **实现方式**
   - 在配置解析阶段合并 cwd 设置
   - 在任务执行阶段应用合并后的 cwd

## Risks / Trade-offs

- **路径解析复杂性**: 用户可能使用相对路径，需要明确基准目录
  - 缓解：在文档中明确说明 cwd 的基准目录
- **向后兼容性**: 现有配置文件缺少 cwd 字段
  - 缓解：所有 cwd 字段均为可选，默认使用当前目录
