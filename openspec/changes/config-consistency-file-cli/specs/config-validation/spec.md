## ADDED Requirements

### Requirement: CLI 验证配置文件和命令行参数的一致性
系统必须在启动时验证配置文件和命令行参数的一致性，检测潜在的冲突或配置错误。

#### Scenario: 配置文件和命令行参数的 cwd 冲突
- **WHEN** 用户在配置文件中指定 `cwd` 并通过 `--cwd` 命令行参数指定不同的路径
- **THEN** 系统使用命令行参数的值（命令行优先）
- **THEN** 系统输出警告信息说明配置冲突

#### Scenario: 配置文件和命令行参数的 agent 冲突
- **WHEN** 用户在配置文件中指定 `agent` 并通过 `--agent` 命令行参数指定不同的代理命令
- **THEN** 系统使用命令行参数的值（命令行优先）
- **THEN** 系统输出警告信息说明配置冲突

#### Scenario: 配置文件不存在
- **WHEN** 用户未提供配置文件且默认配置文件不存在
- **THEN** 系统继续运行，仅使用命令行参数
- **THEN** 系统不输出错误

#### Scenario: 配置文件格式错误
- **WHEN** 配置文件存在但包含无效的 YAML 格式
- **THEN** 系统输出明确的错误信息说明文件位置和解析错误
- **THEN** 系统退出并返回非零状态码

### Requirement: 验证配置值的完整性
系统必须验证配置值是否完整且有效，确保所有必需的配置项都已设置。

#### Scenario: 任务列表为空
- **WHEN** 配置文件中缺少 `tasks` 字段或 `tasks` 为空数组
- **THEN** 系统输出错误信息说明需要提供任务列表
- **THEN** 系统退出并返回非零状态码

#### Scenario: 工作目录不存在
- **WHEN** 配置的 `cwd` 路径不存在
- **THEN** 系统输出错误信息说明路径不存在
- **THEN** 系统退出并返回非零状态码
