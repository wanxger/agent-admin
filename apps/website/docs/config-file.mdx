# 配置文件

Agent Admin 使用 YAML 格式的配置文件来定义任务和参数。

## 配置文件位置

默认配置文件名为 `aa.yaml`，放在项目根目录下。

你也可以使用 `-f` 或 `--file` 参数指定其他位置的配置文件：

```bash
aa -f /path/to/config.yaml
```

## 配置文件格式

### 基本格式

```yaml
# 可选：项目级工作目录
cwd: '/path/to/project'

# 可选：代理命令
agent: 'opencode acp'

# 必需：任务列表
tasks:
  - '任务描述 1'
  - '任务描述 2'
```

### 完整示例

```yaml
cwd: '/home/user/my-project'
agent: 'custom-agent --config /path/to/config'

tasks:
  - task: '构建前端'
    cwd: './frontend'

  - task: '构建后端'
    cwd: './backend'

  - '运行集成测试'
```

## 配置项说明

### cwd

项目级工作目录。可以是绝对路径或相对路径。

**类型**：字符串
**必需**：否
**默认值**：当前目录

```yaml
cwd: '/path/to/project'
```

### agent

代理命令。用于执行任务的命令。

**类型**：字符串
**必需**：否
**默认值**：`opencode acp`

```yaml
agent: 'custom-agent --config /path/to/config'
```

### tasks

任务列表。可以是简单的任务描述字符串，也可以是包含额外配置的任务对象。

**类型**：数组
**必需**：是

#### 简单任务

```yaml
tasks:
  - '任务描述'
```

#### 任务对象

```yaml
tasks:
  - task: '任务描述'
    cwd: '/path/to/task'
```

**任务对象属性**：

- `task`（必需）：任务描述
- `cwd`（可选）：任务级工作目录

## 配置优先级

### 工作目录优先级

1. 命令行 `--cwd`
2. 任务级 `cwd`
3. 项目级 `cwd`
4. 当前目录

### 代理命令优先级

1. 命令行 `--agent`
2. 配置文件 `agent`
3. 默认值（`opencode acp`）

## 配置示例

### 示例 1：简单任务

```yaml
tasks:
  - '构建项目'
  - '运行测试'
```

### 示例 2：指定工作目录

```yaml
cwd: '/home/user/my-project'

tasks:
  - '初始化项目'
  - '实现核心功能'
  - '编写测试文档'
```

### 示例 3：任务级工作目录

```yaml
tasks:
  - task: '构建前端'
    cwd: './frontend'

  - task: '构建后端'
    cwd: './backend'

  - '运行集成测试'
```

### 示例 4：自定义代理

```yaml
agent: 'my-agent --verbose --timeout 300'

tasks:
  - '优化数据库查询'
  - '实现缓存层'
  - '添加日志记录'
```

### 示例 5：完整配置

```yaml
cwd: '/home/user/my-project'
agent: 'custom-agent --config /path/to/config'

tasks:
  - task: '设置前端环境'
    cwd: './frontend'

  - task: '设置后端环境'
    cwd: './backend'

  - '创建数据库迁移'
  - '配置 CI/CD'
  - '编写部署文档'
```

## 配置验证

Agent Admin 会在启动时验证配置：

- ✅ YAML 语法正确
- ✅ 工作目录存在
- ✅ 任务列表不为空

如果配置无效，Agent Admin 会显示错误信息并退出。

## 常见问题

### Q: 如何在不同环境使用不同配置？

A: 创建多个配置文件，使用 `-f` 参数指定：

```bash
# 开发环境
aa -f config/dev.yaml

# 生产环境
aa -f config/prod.yaml
```

### Q: 如何在配置文件中使用环境变量？

A: 目前不支持直接在配置文件中使用环境变量。你可以使用命令行参数覆盖：

```bash
aa -c $PROJECT_DIR
```

### Q: 任务执行顺序是怎样的？

A: 任务按照配置文件中定义的顺序依次执行。
