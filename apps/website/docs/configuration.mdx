# 配置

Agent Admin 支持通过命令行参数和配置文件两种方式进行配置。

## 命令行参数

### 基本参数

| 参数         | 简写 | 说明                 |
| ------------ | ---- | -------------------- |
| `--task`     | `-t` | 指定要执行的任务描述 |
| `--cwd`      | `-c` | 指定工作目录         |
| `--agent`    | -    | 指定代理命令         |
| `-f, --file` | -    | 指定配置文件路径     |

### 示例

```bash
# 指定任务
aa -t "创建用户认证系统"

# 指定工作目录
aa -c /path/to/project -t "构建项目"

# 指定代理命令
aa --agent "custom-agent --config /path/to/config" -t "优化代码"

# 指定配置文件
aa -f /path/to/config.yaml
```

## 配置文件

配置文件使用 YAML 格式，默认文件名为 `aa.yaml`。

### 配置文件结构

```yaml
# 项目级工作目录（可选）
cwd: '/path/to/project'

# 代理命令（可选）
agent: 'opencode acp'

# 任务列表（必需）
tasks:
  - '任务描述 1'
  - task: '任务描述 2'
    cwd: '/path/to/task2'
```

### 配置优先级

1. **命令行参数**（最高优先级）
2. **配置文件**
3. **默认值**

### 工作目录优先级

1. 命令行 `--cwd`
2. 任务级 `cwd`
3. 项目级 `cwd`
4. 当前目录

### 代理命令优先级

1. 命令行 `--agent`
2. 配置文件 `agent`
3. 默认值（`opencode acp`）

## 环境变量

目前 Agent Admin 不支持通过环境变量配置，所有配置都通过命令行参数或配置文件完成。

## 配置验证

Agent Admin 会在启动时验证配置：

- 配置文件格式（YAML 语法）
- 工作目录是否存在
- 任务列表是否为空

如果配置无效，Agent Admin 会显示错误信息并退出。

## 高级用法

### 条件配置

你可以在不同的环境使用不同的配置文件：

```bash
# 开发环境
aa -f config/dev.yaml

# 生产环境
aa -f config/prod.yaml
```

### 组合使用

配置文件和命令行参数可以组合使用，命令行参数会覆盖配置文件中的值：

```yaml
# aa.yaml
cwd: '/default/path'
tasks:
  - '任务 1'
```

```bash
# 使用配置文件，但覆盖工作目录
aa -c /custom/path
```
